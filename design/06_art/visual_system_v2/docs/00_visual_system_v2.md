# 新版视觉系统 v2：一套骨架，四种表现模式

## 1. 设计结论

四张概念图都值得保留，但不能把它们当成四套平行资产生产。正确做法是建立一个稳定的 `PresentationProfile`：剧情、玩法、碰撞、文字内容与输入焦点完全不变；profile 只决定图像如何画出来。

```text
Game State
  ├─ location / event / actors / dialogue / combat
  └─ presentation_profile = A | B | C | D
       ├─ panel skin
       ├─ palette polarity
       ├─ dither budget
       ├─ portrait crop
       └─ VFX vocabulary
```

这样做的直接收益：

- 剧情脚本不需要为四种 UI 重写；
- 同一角色动画可以在普通、密集、木版和反相模式复用；
- D 模式不需要再画一套碰撞或弹幕；
- 可以先完整量产 A，再逐步增加 B／C／D，而不会阻塞游戏本体；
- 后续如果删除或收缩某种皮肤，不会破坏存档与事件数据。

## 2. 共用硬规格

| 项目 | 锁定值 | 原因 |
|---|---:|---|
| 内部画布 | 320×180 | 16:9，能容纳探索、弹幕、格斗与三行英文对话 |
| 放大 | 2×／3×／4×整数倍 | 避免像素宽度不一致 |
| 采样 | nearest-neighbor | 禁止任何平滑插值 |
| UI 网格 | 4 px | 面板、留白、焦点矩形可以共享 token |
| 场景 tile | 16×16 px | 20×11.25 tile 的探索视野 |
| 可见调色板 | 黑／白 | alpha 只用于精灵合成，不引入灰色 |
| 网点 | 4×4 Bayer | 25／50／75% 三档，可复现且不随机闪烁 |
| Model S | 16×24 | 地图、群众、印章式小人 |
| Model M | 24×32 | 主要探索与小游戏 |
| Model L | 32×48 | 格斗、戏剧特写、关键动作 |
| 对话立绘 | 80×104 或 88×120 | 精确像素绘制，不放大普通精灵 |

## 3. 四种 profile 的职责

### A — Pocket Shrine

这是默认模式，也是生产基线。白底黑墨，黑色覆盖率约 25–35%。角色外轮廓 2 px，远景允许 25% 网点，前景尽量使用完整黑块与白色切口。

使用场景：

- 博丽神社与人间之里的普通探索；
- 普通室外、商店、日志与设置；
- 大部分日常对话；
- 新玩家第一次遇到新机制时。

禁止：为了“复古”给每个空白处加噪点；用 1 px 长线画所有建筑；把字体放大到比角色还抢眼。

### B — PC-98 Dither

这是高密度模式。仍然只有黑白，但使用 25／50／75% 有序网点来暗示材质和传统色。面板有双框和角标，信息更密，角色与立绘可以稍多细节。

使用场景：

- 红魔馆图书馆、时钟走廊、厨房与复杂机关；
- 永远亭药柜、月都相关室内；
- 关键立绘、长对话、调查推理；
- 需要“古早电脑作品”味道的章节。

禁止：在移动弹幕背后使用密集 50% 网点；把 B 当作全游戏默认导致视觉疲劳；通过灰色抗锯齿伪造网点。

### C — Woodblock Adventure

这是叙事模式。面板像纸条、印章、折角与木版墨块，布局允许不对称，立绘可越过边框最多 8 px。情绪主要依靠大片黑白形状、方向性排线和留白，而不是渐变。

使用场景：

- 路线章节卡与关系阈值；
- 亲密但克制的对话；
- 回忆、传闻、历史重述；
- 白玉楼、幽灵与季节余韵；
- 结局前后的安静 afterbeat。

禁止：把纸纹做成连续灰色照片；所有普通对话都破框；在弹幕进行中使用大面积不对称选择条。

### D — Midnight LCD

这是战斗／异常模式。默认黑底白像素，黑色覆盖率 70–85%。HUD 变薄，弹幕区域禁止细密网点。精灵可使用反相版本；关键命中用 1 帧局部反相，而不是光晕。

使用场景：

- 夜间竹林、地下、梦境、月面与异常空间；
- 弹幕、符卡、格斗高潮；
- route threshold 的危险一侧；
- 需要强烈剪影或电子液晶感的小游戏。

禁止：霓虹光、CRT 模糊、全屏扫描线；在阅读长对话时持续黑底；用闪烁代替清晰的危险边缘。

## 4. Profile 切换规则

1. 普通状态默认 A。
2. 进入被标记为 `dense_interior` 的场所时，可在场景淡出后切至 B。
3. 事件节点标记 `narrative_plateau`、`memory` 或 `route_threshold` 时，可切至 C。
4. `combat_night`、`danmaku`、`dream_danger`、`underground` 可切至 D。
5. 一帧内只能有一个主 profile；不得把 A 面板、B 背景、C 立绘边框和 D HUD 任意拼接。
6. profile 切换发生在：场景切换、纸张 wipe、黑白 iris 或 6 帧以内的边框折叠。
7. 玩家可在可访问性设置中强制 A：所有内容必须在 A 下仍可完成。

## 5. 视觉层级

每帧只允许三个主要层级：

1. **交互层**：角色、门、道具、弹幕、选择焦点；最高对比度；
2. **场景层**：建筑与可行走面；使用完整轮廓和有限纹理；
3. **气氛层**：天空、雾、远树、记忆污染；允许网点，但不得与交互层竞争。

在 1× 下快速眯眼检查时，玩家应先看见角色与互动目标，再看见道路／平台，最后才看见远景纹理。

## 6. 黑白面积预算

| Profile | 黑色面积建议 | 典型用途 |
|---|---:|---|
| A | 25–35% | 清爽、贫而有呼吸感的神社与室外 |
| B | 40–50% | 书架、时钟、帷幕、药柜等密集室内 |
| C | 不求平均；使用大块交替 | 木版构图、情绪留白、章节卡 |
| D | 70–85% | 夜战、月亮切口、白色弹幕与精灵 |

这些比例不是逐帧硬门槛，而是用来防止 A 再次退化为空白线框，也防止 B／D 变成不可读的黑团。

## 7. 与概念图的关系

`assets/concepts/` 中的 A／B／C／D 四张图负责回答“目标气质是什么”；`assets/sprites/`、`assets/tiles/` 与 `assets/ui/` 回答“在真实网格里怎么做”。概念图可以有更丰富的角色细节和构图，但量产资产必须通过尺寸、调色板、锚点与 1× 可读性检查。

2. 通过 Reimu／Marisa／Sakuya 的完整 M 动画与两张对话立绘；
3. 用同一数据切换 D，验证弹幕与反相精灵；
4. 为红魔馆扩展 B tile 与 Sakuya／Patchouli 立绘；
5. 用 C 制作一个 60–90 秒的关系阈值事件；
6. 通过后再扩展角色，而不是先批量画几十名角色。
