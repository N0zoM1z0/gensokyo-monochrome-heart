# 三组基础 Tile 与地区语法

## 1. Atlas 结构

每组基础 atlas 为 `128 × 64`，即 8 列 × 4 行，每格 `16 × 16`。三组 atlas 的格位语义相同，便于地图工具、碰撞数据和材质音效复用。

| 行 | 索引 | 内容 |
|---|---|---|
| 0 | 0–7 | 天空／空白、地面顶边、木板、石材、墙、屋顶、窗／屏风、台阶 |
| 1 | 8–15 | 柱、栅格、地标、观察闪光、容器、灯、传闻纸条、关系道具 |
| 2 | 16–23 | 25／50／75% 网点、天气、记忆框、前景遮罩、危险、出口 |
| 3 | 24–31 | 地区专属：建筑、地标、材质与机关 |

原型中最后三个格位保留给后续地区事件，不应用通用装饰填满。保留空位比每个格子都塞素材更利于长期生产。

## 2. 通用碰撞规则

- 实体地面的最上边使用连续 2 px 黑线；
- 非碰撞装饰不得复用连续 2 px 顶边；
- 单向平台使用 `2-1` 断线节奏；
- 门、梯、出口必须拥有不同的内部负形；
- 前景遮罩与碰撞完全分离；
- 地图编辑器读取 atlas metadata，不从像素边界自动推断碰撞；
- 交互范围从对象原点导出，不使用整张 tile 的矩形。

## 3. 交互形状语言

| 类别 | 图形规则 | 目的 |
|---|---|---|
| Observe | 2×2 十字闪光，周围留白 | 一眼看出可调查，不像弹幕 |
| Carry | 容器底部双把手／双缺口 | 与普通箱子区分 |
| Repair | 三段斜裂纹 | 不与装饰排线混淆 |
| Danger | 交替反相边缘 | 不依赖颜色警示 |
| Rumor | 带折角的纸片 | 与 Journal 视觉一致 |
| Memory | 不完整矩形框 | 表示缺失／被改写而非普通门 |
| Companion skill | 对应角色的 9×9 印章 | 技能入口不长期占 HUD |

## 4. 博丽神社 tile

### 视觉名词

- 长水平缘侧；
- 磨损木板；
- 赛钱箱竖条；
- 注连绳与纸垂；
- 古树与空旷天空；
- 贫穷但有呼吸空间。

### 网点预算

神社默认使用 A。天空或远树可用 25% 网点；木板靠线的方向与间距表现，不使用 50% 大面积铺底。树干是完整黑块加白色纵向切口。

### 首批 tile

- 缘侧木板与落差边；
- 障子／格门；
- 屋瓦节奏；
- 鸟居；
- 注连绳／纸垂；
- 古树干；
- 赛钱箱；
- 石灯笼；
- 观察闪光、茶杯、坐垫、扫帚等事件道具。

### 组合测试

神社场景必须留出至少一个连续 80 px 的低纹理区域，让角色、杯子与对话选择可读。不要因为“画面太空”而把天空填满云和鸟。

## 5. 红魔馆 tile

### 视觉名词

- 高门、拱窗；
- 书架重复；
- 棋盘地面；
- 时钟、烛台、帷幕；
- 厨房的硬直角；
- 重复中有一个微小错误。

### 网点预算

红魔馆主要使用 B。25% 表示远墙或布面，50% 表示中景木材／绒布，75% 仅表示深凹槽或“红”的暗示。角色脸、刀、交互纸条和移动弹幕后方禁止网点。

### 首批 tile

- 书架与不同高度书脊；
- 拱窗与月光切口；
- 棋盘地板；
- 钟面与钟摆；
- 双框门；
- 烛台；
- 厚帷幕；
- 图书馆梯、药瓶、茶车与刀具架。

### 重复异常

红魔馆可以利用 tile 重复制造不安：连续六个相同书架中只改变一个书脊、一个钟面或一个门把。变化必须是事件数据，而不是随机噪声。

## 6. 永远亭／迷途竹林 tile

### 视觉名词

- 垂直竹节；
- 大月亮圆形；
- 障子与相似走廊；
- 药柜与抽屉；
- 榻榻米格；
- 同类 tile 只偏移 1–2 px 造成迷路感。

### 网点预算

普通室内可用 A 或 B；夜间竹林与弹幕切换 D。D 的活动弹幕区不得铺 50% 网点，竹林靠粗白线、黑底与月亮切口建立层次。

### 首批 tile

- 竹节三种偏移；
- 竹叶对角线；
- 月亮／残月；
- 障子与走廊；
- 榻榻米；
- 药柜／抽屉；
- 诊疗灯、药瓶、兔耳路标；
- 月面干涉线与记忆矩形。

### 迷路规则

相似走廊可共享 80% tile，但必须有可学习的差异：竹节高度、月亮位置、药柜把手、脚步声或关系道具。迷路不能只靠完全相同的画面与随机传送。

## 7. Dither 使用

4×4 Bayer 网点固定相位，不随镜头移动重新对齐：

- 25%：远景、薄雾、浅材质；
- 50%：中景大材质，静态画面谨慎使用；
- 75%：深凹槽、危险暗示、隐含“红／紫”；
- 全屏移动 checker 禁止；
- 角色脸、8px 字体、细弹幕、碰撞提示禁止；
- 动画对象的网点相位绑定对象，而不是绑定屏幕，避免闪烁。

## 8. 每个 Spot 的最低场景资产

每个 spot 至少需要：

- 一件随章节变化的物品；
- 一件关系专属道具；
- 一处日常维护细节；
- 一处与 incident 矛盾的细节；
- 一张安静 afterbeat 构图；
- calm 与 incident 两种 profile／极性测试；
- 不超过三种同时播放的粒子。
