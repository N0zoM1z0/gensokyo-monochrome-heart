# 像素字体与日英本地化

## 1. 目标

字体必须服务于 320×180，而不是把普通矢量字体缩小后强行锐化。英文和日文共用组件矩形，但拥有不同字格、换行和禁则规则。

## 2. 英文：Kiri-8 原型

本 demo 的英文像素字形是 `5 × 7` 主体，放在 `6 × 8` 的实际推进单元中；UI token 仍按 8 px 模块对齐。特点：

- 纯黑／纯白；
- 每字符 1 px tracking；
- 大写优先；
- 不做矢量描边或 faux bold；
- 适合短标签、HUD 与原型对话；
- 字母 `I／L／1`、`O／0` 在上下文中仍需注意。

正式长篇英文对话可补充小写字形，但不应为了“像素”牺牲阅读速度。若加入小写，x-height 至少 5 px。

## 3. 日文：DotGothic16 阈值化原型

本 demo 附带 DotGothic16 的日文与拉丁 WOFF2，许可证为 SIL Open Font License 1.1。它是可缩放字体，不是原生 BDF 位图；当前用途是：

1. 在 12 px 逻辑尺寸渲染；
2. 关闭平滑缩放；
3. 用 50% 阈值二值化；
4. 在 1× 和整数倍下检查笔画堵塞；
5. 验证日文行宽、禁则与名字标签。

如果后续选用原生位图 JIS 字体，必须确认：商业／同人发行许可、再分发许可、覆盖字形、标点、假名、常用汉字、拉丁与数字风格。

## 4. 字格与行距

| 场景 | 英文 | 日文 |
|---|---|---|
| HUD 标签 | 6×8 推进；7–8 px 字号 | 10–12 px 字格 |
| 人名标签 | 6×8 推进；最多 12–14 字符 | 12 px 字格；宽度可扩至 72 px |
| 对话正文 | 8–9 px 行高；3 行 | 12–13 px 行高；最多 4 行 |
| 选择项 | 12 px 行高 | 12–13 px 行高 |
| 日志标题 | 8–10 px | 12–14 px |

名字标签在日文中先扩宽，最后才减小字号；不允许通过非整数横向压缩塞入。

## 5. 换行与禁则

- 按 grapheme cluster 处理，不按 UTF-8 字节切分；
- 日文启用行首／行末禁则；
- `、。！？）」』】` 不得出现在行首；
- `（「『【` 不得留在行末；
- 英文不拆开缩写、数字单位和专名；
- ruby 不跨行；
- 角色名、符卡名和地点名可通过作者标记提供优先断点；
- 自动模式按语言与字符数估算阅读时间，不推进选择项。

## 6. 文本语气与 UI 大小写

- 英文 HUD 可全大写；长对话不建议全大写；
- 日文避免为了“复古”滥用半角片假名；
- `DIRECT／PLAYFUL／PATIENT／DEFIANT` 是意图标签，正文仍显示实际行动；
- 日文标签需由本地化重新写为语气／行动，不逐字硬译；
- 重要名词使用 9×9 印章或 glossary mark，不使用彩色文字。

## 7. 反相与可读性

D 模式白字黑底时：

- 禁止 1 px 细字体叠在竹叶或弹幕后；
- HUD 文字至少离边框 3 px；
- 选择焦点使用粗框／反相块，不只用闪烁；
- 长对话优先临时打开白底黑字纸条，避免持续反相阅读；
- 字形阈值与普通模式分别检查，不能假定简单颜色反转一定清楚。
