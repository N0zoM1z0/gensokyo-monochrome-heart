# Definition of Done

## A code task is done when

- implementation follows layer boundaries;
- public APIs are typed and documented;
- tests cover success and failure paths;
- relevant validators pass;
- no new unapproved dependency;
- no steady-state avoidable allocation in hot loops without justification;
- error paths are user-safe and diagnosable;
- debug code is channel-gated;
- design decision recorded if a non-negotiable changed;
- exact verification is reported.

## A content event is done when

- event graph validates and all ends are reachable as intended;
- EN and JA text complete and reviewed;
- character `skills.md` and relationship edges reviewed;
- Canon/Fanon/Original origin tags assigned;
- fanon dial within ceiling;
- protagonist is not made universally central to every character motive;
- choice tones are distinct and not disguised “right/wrong” buttons;
- clear, loss, and assist outcomes authored;
- quiet afterbeat exists where designed;
- comfort alternatives preserve information;
- Journal observation is accurate;
- save/replay tests pass;
- required assets have provenance.

## A combat encounter is done when

- mechanic expresses character/location;
- telegraphs meet Story-mode rule;
- density/speed/contrast assists tested;
- deterministic replay fixture exists;
- phase retry and accept-loss work;
- performance captured;
- warnings have visual/audio alternatives;
- no official pattern or frame-data copy;
- event return is typed and idempotent.

## A milestone is done when

- every acceptance item is pass or explicitly waived by owner with rationale;
- clean import/build succeeds;
- tests and validators archived;
- EN/JA screenshots archived;
- accessibility matrix complete;
- performance report complete where relevant;
- known issues prioritized;
- save/replay compatibility considered;
- asset/license scan complete;
- documentation updated;
- producer accepts the playable result, not only code diff.

## A release candidate is done when

- fan-work and storefront rules rechecked;
- unofficial notice and credits correct;
- no official/ripped/unlicensed asset;
- all permissions archived;
- all P0 tests pass;
- supported platforms pass clean-machine install;
- save migration and rollback tested;
- privacy/logging reviewed;
- minimum hardware claims measured;
- SHA-256 manifest created;
- previous build retained for rollback;
- support contact and known issues prepared.
