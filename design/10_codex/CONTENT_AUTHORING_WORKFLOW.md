# Content Authoring Workflow

## 1. Event pitch

Writer submits:
- region/spot;
- lead and support characters;
- contradiction or mundane problem;
- character non-romantic motive;
- comedy escalation;
- mechanical expression;
- quiet afterbeat;
- route/main-plot changes;
- canon/fanon/original estimate;
- comfort tags;
- asset/audio needs;
- estimated duration.

## 2. Character check

Read every visible character's `skills.md` and relevant relationship edges. Record:
- immediate motive;
- what each character will not say directly;
- what each character can refuse;
- fanon ceiling;
- one autonomy beat unrelated to protagonist;
- likely EN and JA rhythm risks.

## 3. Graybox graph

Build event graph with placeholder localization keys and geometric actions. Validate all branches before writing polished dialogue. Mechanical mode uses fixture result first.

## 4. Dialogue draft

Draft intent and nonverbal cue before line text. Write EN and JA as parallel localizations. Agent proposals may assist but are not accepted automatically.

## 5. Mechanical integration

Specify:
- mode context;
- controls;
- tutorial;
- assists;
- clear/excellent/loss result tags;
- state-independent retry;
- return branches;
- performance and asset budget.

## 6. Review gates

1. narrative structure;
2. character/canon;
3. boundaries/comfort;
4. gameplay communication;
5. EN edit;
6. JA edit;
7. localization layout;
8. playtest;
9. provenance/license;
10. final state/replay audit.

## 7. Revision discipline

Changing a line should not silently change event effects. Changing an effect requires state and route review. Changing a mechanic result tag requires branch validation and save/replay review.

## 8. Completion packet

Each event directory contains:
- event JSON;
- dialogue/localization rows;
- fixture state;
- expected branch map;
- screenshots EN/JA;
- replay if mechanical;
- review checklist;
- asset/audio references;
- source/provenance notes.
