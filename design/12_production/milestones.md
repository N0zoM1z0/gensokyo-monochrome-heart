# Production Milestones

Durations are indicative for a small experienced part-time/full-time hybrid team. They are not promises. Re-estimate after the vertical slice.

## Phase A — Preproduction lock (2–4 weeks)

Outputs:
- engine/version and repository lock;
- vertical-slice content lock;
- protagonist presentation decision;
- Model S/M/L art prototypes;
- one music/SFX test cue;
- EN/JA terminology register;
- legal/provenance workflow;
- task estimates and owners.

Gate:
- no unresolved question blocks the Empty Cushion slice;
- scope owner signs vertical-slice acceptance list.

## Phase B — Technical foundation (4–8 weeks)

Outputs:
- M00–M04;
- shell, input, content DB, state/save, event/dialogue;
- headless validation and CI;
- geometric placeholder art only.

Gate:
- sample event executes all branches with mocked mode results;
- save migration fixture passes;
- EN/JA UI shell passes.

## Phase C — Vertical slice mechanics (8–14 weeks)

Outputs:
- Shrine exploration;
- Tea Temperature minigame;
- boundary-stain danmaku;
- compact Reimu/Marisa duel;
- Journal/afterbeat;
- adaptive test-tone music;
- accessibility presets.

Gate:
- all P0 vertical-slice acceptance items pass;
- five external playtests complete;
- performance baseline measured.

## Phase D — Tooling and second-region proof (6–10 weeks)

Outputs:
- authoring tools;
- Scarlet Devil Mansion slice;
- Sakuya missing-minute event;
- first production-quality art/audio pipeline items.

Gate:
- no region/character hard-code in generic systems;
- nonprogrammer can author and preview an event;
- art/audio provenance workflow demonstrated.

## Phase E — Main campaign backbone (4–8 months)

Outputs:
- five headline regions;
- six chapters through playable graybox/content-complete state;
- 28 main events plus support chains;
- main Archive finale prototype;
- first six deep routes in content-complete state.

Gate:
- main story can be completed in Story preset;
- every chapter has EN/JA first editorial pass;
- saves remain migratable.

## Phase F — Route and roster expansion (4–8 months)

Outputs:
- all 12 deep routes;
- support roster integration;
- seasonal/postgame framework;
- eight launch fighters / ten danmaku loadouts if scope remains healthy;
- production art/audio for headline content.

Gate:
- route checklists complete;
- canon/fanon review pass;
- no mandatory high-skill block.

## Phase G — Alpha (2–4 months)

Definition:
- feature complete;
- content substantially complete;
- placeholders only where tracked;
- all routes and ending families accessible;
- full-game save/replay works.

Focus:
- bug fixing, pacing, balance, localization, accessibility, missing assets.

## Phase H — Beta / content lock (2–3 months)

Definition:
- no new features;
- text and asset lock except fixes;
- complete rights/provenance records;
- performance/stability/accessibility hardening;
- clean-machine builds.

## Phase I — Release candidate (4–8 weeks)

Outputs:
- signed checklist;
- Windows/Linux exports, optional macOS;
- demo and compatibility policy;
- release notes, known issues, support packet;
- hash manifest and rollback build.

## Stop/go criteria

At the end of Phases C, D, and E, the producer must choose:
- continue current scope;
- reduce route/roster/mechanical scope;
- extend schedule/budget;
- pause production.

Do not silently absorb overruns through unpaid crunch.
