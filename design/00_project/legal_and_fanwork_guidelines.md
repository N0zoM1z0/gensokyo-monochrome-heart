# Fan-Work, Rights, and Release Checklist

This document is a production reminder, not legal advice. Check the current official Touhou Project fan-work guidelines at each public milestone.

## Mandatory asset policy

Do not ship:
- extracted official sprites, backgrounds, sound effects, music files, endings, fonts, or code;
- screenshots used as game assets;
- another fan creator's arrangement, illustration, translation, or design without permission;
- a title treatment that could be mistaken for an official Touhou release.

Create or license:
- all code;
- all character and environment pixel art;
- all sound effects;
- all UI assets;
- all arrangements or original music;
- all English and Japanese text.

## Presentation

The title screen, store page, README, and credits should clearly state that this is an unofficial Touhou Project fan work.

Suggested line:

> This is an unofficial Touhou Project fan work. Touhou Project is created by Team Shanghai Alice.

## Music

A theme title may be listed as an arrangement reference in preproduction. Shipping an arrangement still requires:
- a new performance/production or a properly licensed recording;
- compliance with the current fan-work rules;
- correct composer and source-game credit;
- verification of platform and distributor terms.

## Distribution checks

Before release:
- review the official guidelines again;
- confirm the chosen storefront or indie-game platform is permitted;
- confirm monetization, pricing, demo, and crowdfunding plans;
- replace every placeholder with an original or licensed asset;
- run a dependency and license audit;
- keep a permission archive for guest work;
- provide a contact address for rights questions.

## Project-specific conservative rule

Because this game contains romance and many characters with ambiguous ages, the base release remains non-explicit and avoids sexualized marketing. This is both a creative choice and a risk-reduction measure.
